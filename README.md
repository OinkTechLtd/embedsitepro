# EmbedSitePro

**Чистый плеер с любого сайта — без рекламы и мусора.**

Сервис находит видеоплеер на странице и открывает его в полноэкранном режиме, убирая всё лишнее: баннеры, навигацию, попапы и рекламу.

---

## Структура проекта

```
EmbedSitePro/
├── index.html      — Лендинг (главная страница)
├── player.html     — Страница плеера
├── player.js       — Логика поиска и запуска плеера
├── styles.css      — Стили (лендинг + плеер)
└── README.md       — Документация
```

## Как запустить

### Вариант 1 — Просто открыть файл
Открой `index.html` в браузере. Всё работает без сервера.

> ⚠️ Некоторые CORS-прокси могут не работать на `file://`.
> Рекомендуется локальный сервер.

### Вариант 2 — Локальный сервер (рекомендуется)

**Python:**
```bash
cd EmbedSitePro
python3 -m http.server 8080
# Открой http://localhost:8080
```

**Node.js (npx serve):**
```bash
cd EmbedSitePro
npx serve .
```

**VS Code:** установи расширение Live Server, правый клик на `index.html` → Open with Live Server.

## Как это работает

### Стратегии извлечения плеера (по порядку):

1. **Прямой iframe** — пробуем встроить исходный URL в `<iframe>`. Если сайт разрешает — сразу показываем.

2. **CORS-прокси** — загружаем HTML страницы через бесплатные прокси:
   - `api.allorigins.win`
   - `corsproxy.io`
   - `api.codetabs.com`

3. **Парсинг HTML** — ищем в полученном HTML:
   - `.m3u8` потоки (HLS)
   - `.mp4` файлы
   - `<iframe src="...">` — плееры
   - `<embed src="...">` — встроенные плееры
   - `data-src`, `data-url` атрибуты
   - JS-переменные: `file:`, `src:`, `stream:`

4. **HLS плеер** — если найден `.m3u8`, используем `hls.js` для воспроизведения.

5. **Нативное видео** — если найден `.mp4`, используем `<video>`.

6. **Iframe плеера** — если найден сторонний iframe — встраиваем только его.

## Почему некоторые сайты не работают

| Причина | Что происходит |
|---------|---------------|
| `X-Frame-Options: DENY` | Браузер блокирует iframe — пробуем CORS-прокси |
| `Content-Security-Policy` | Та же проблема — тот же обход |
| Плеер загружается через JS | Парсинг не видит плеер — нужен headless браузер |
| Требует авторизацию | Поток недоступен без куки/токена |
| DRM защита | Нельзя воспроизвести без лицензии |

## Расширение (для разработчиков)

Чтобы добавить поддержку сложных сайтов, можно добавить backend на Node.js с Puppeteer:

```js
// Псевдокод для серверного рендеринга
const browser = await puppeteer.launch();
const page = await browser.newPage();
await page.goto(url);
// Перехватываем сетевые запросы к .m3u8
page.on('request', req => {
  if (req.url().includes('.m3u8')) console.log(req.url());
});
```
